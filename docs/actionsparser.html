actionsparser #

actions

are text based representatsions of actions which need to be executed

example

!!tflibrary.booksmanager.book_add 
    gitsource:'books'
    path:'technology/src'
    name:technology

the first one is the action, the rest are the params

fn new #

fn new(args ActionParserArgs) !ActionsParser

get an actionparser and do the processing, can always add text and files later see actionparser.path_add and text_add it will sort the actions if filter is given and remove the ones which are in error or not belonging to book or actor params text:string path:string //can be dirpath or filepath filter:[]string //how to sort & filter actor:string book:string

enum ActionState #

enum ActionState {
	init // first state
	next // will continue with next steps
	restart
	error
	done // means we don't process the next ones
}

struct ActionParserArgs #

struct ActionParserArgs {
pub:
	text   string
	path   string   // can be dir or file
	filter []string // how to sort & filter
	actor  string
	book   string
}

struct ActionsParser #

struct ActionsParser {
pub mut:
	unsorted []Action // should be empty after filter action
	ok       []Action // the ones ready to be processed they are in order
	error    []Action // the ones who are in error
	skipped  []Action // the ones which won't be processed because its not right book or even actor
	actor    string
	book     string
	filter   []string
}

fn (mut actions ActionsParser) filter() !

make sure that only actions are remembered linked to the actor or book and also sorted in right order
